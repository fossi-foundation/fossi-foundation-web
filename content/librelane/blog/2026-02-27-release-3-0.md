The LibreLane team is proud to finally release **LibreLane 3.0**.

This is the second major version release for LibreLane and our first outside of
Efabless. It has been in development for quite some time and is packed with
new features, enhancements, support for new technologies, and more, and our
first API cleanup (which caused the major version bump).

Some highlights are as follows:

- A new flow named `Chip`, which not only hardens macros, but includes pad ring
  generation  for top-level chips.
  - Created new steps: `KLayout.Filler`, `KLayout.SealRing`, `KLayout.Density`,
    and more, to support this new flow.
- Support for the [ihp-sg13](https://github.com/IHP-GmbH/IHP-Open-PDK) PDK
  family, which you can use to tape-out chips on the IHP shuttles using
  [this template](https://github.com/IHP-GmbH/ihp-sg13g2-librelane-template). 
  family.
- A refreshed and verified tape-out flow for the gf180mcu PDK, which you can use
  to tape-out chips on the [wafer.space](https://wafer.space) shuttles using
  [this template](https://github.com/wafer-space/gf180mcu-project-template).
- Improved SystemVerilog support through the
  [yosys-slang](https://github.com/povik/yosys-slang) plugin.
- Reworked `DesignFormat`s from an enumeration to a factory format similar to
  steps and flows, allowing plugins to add new design formats.
- Updated the Nix environment to NixOS 25.11, as well as all the supporting
  tools for the `Classic` and `Chip` flows.
- Updated the Nix recipes for LibreLane environments to support loading
  user-defined Yosys plugins and add user-defined Python packages to
  Yosys/OpenROAD Python interpreters.
- Better test coverage

In addition to many small improvements, such as improved rendering of the chip
layout, support for non-default rules, reduced file size for both Magic and
KLayout GDS files, better subprocess formatting, and many more improvements and
bug fixes. See the full release notes here:
https://github.com/librelane/librelane/blob/3.0.0/Changelog.md
